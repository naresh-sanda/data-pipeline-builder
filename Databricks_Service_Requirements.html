Service Requirements & API Guide

Version Compatibility Matrix

The following API and Driver versions are REQUIRED when both Source and Target are Databricks (Lakehouse Federation / Intra-workspace).

Required API Versions
  • Jobs API: 2.1 (For repair run/task orchestration)
  • SQL Execution: 2.0 (Warehouse-less execution)
  • Unity Catalog: 2.1 (Lineage & Schema browsing)
JDBC Driver Requirements
  • Class: com.databricks.client.jdbc.Driver
  • Min Version: 2.6.38 (Critical for OAuth support)
  • Recommended: 2.6.42+ (Arrow serialization)

1. Source & Connection Management

Goal: Validate credentials and connectivity to Databricks workspaces.

Type Command / Endpoint Description
REST API GET /api/2.0/clusters/list Connectivity Check: Verifies token validity and workspace reachability.
JDBC/SQL SELECT 1 Warehouse Check: Verifies SQL Warehouse checks and query execution via JDBC.
REST API POST /api/2.0/secrets/scopes/create Security: Creates a secret scope to store credentials (if not using direct pass-through).

2. Metadata Selection (Unity Catalog)

Goal: Browse hierarchy (Catalog > Schema > Table) to select source/target.

Type Command / Endpoint Description
REST API GET /api/2.1/unity-catalog/catalogs Lists available catalogs (needs browse permission).
JDBC/SQL SHOW CATALOGS Standard SQL equivalent for listing catalogs.
REST API GET /api/2.1/unity-catalog/tables Lists tables in a schema.
Params: catalog_name, schema_name
JDBC/SQL SHOW TABLES IN catalog.schema SQL equivalent for listing tables.

3. Target Table Definition

Goal: Define the target table location, storage format (Delta), and constraints.

Type Command / Endpoint Description
REST API POST /api/2.0/sql/statements Executes DDL to create/replace target tables.
CREATE TABLE IF NOT EXISTS ... USING DELTA
JDBC/SQL CREATE TABLE ... Standard DDL for table creation.
REST API GET /api/2.1/unity-catalog/tables/{name} Checks if target table already exists to prompt for overwrite/append logic.

4. Transformation & Preview

Goal: Validate transformation logic and preview data.

Type Command / Endpoint Description
REST API POST /api/2.0/sql/statements Executes SQL for preview.
Body: { "statement": "SELECT * ... LIMIT 10" }
JDBC/SQL SELECT ... LIMIT 10 Direct JDBC execution for data preview.

5. Version Control (Review)

Goal: Commit and push generated SQL/Code to the Git repository.

Type Command / Endpoint Description
REST API GET /api/2.0/repos/{repo_id} Checks status of connected repo branch.
REST API POST /api/2.0/workspace/import Writes the generated pipeline notebook/SQL file to the workspace.

6. Deploy & Create Workflow

Goal: Final step of wizard - Converts pipeline config into a Databricks Job.

Type Command / Endpoint Description
REST API POST /api/2.1/jobs/create Create Workflow: Creates the multi-task Job definition with all dependencies.
REST API POST /api/2.1/jobs/run-now Execute: Immediately triggers the newly created job.
REST API POST /api/2.1/jobs/reset Update: If job exists, updates the definition (idempotent deploy).
