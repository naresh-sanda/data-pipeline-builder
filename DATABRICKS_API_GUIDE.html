Databricks ELT Platform
API Integration Guide

Overview

This document outlines the specific Databricks REST APIs required to power the features of the Databricks ELT Platform application.

1. Connection & Authentication

Goal: Verify connection to the Databricks Workspace.

Operation Method Endpoint Description
Verify Token GET /api/2.0/current-user/me Validates the Personal Access Token (PAT) and retrieves current user info.
List Clusters GET /api/2.0/clusters/list (Optional) Used to check for available compute resources for JDBC/ODBC connections.

2. Metadata Management (Unity Catalog)

Goal: Browse Catalogs, Schemas, and Tables for the "Data Catalog" and "Metadata" steps.

Operation Method Endpoint Description
List Catalogs GET /api/2.1/unity-catalog/catalogs Retrieves a list of all catalogs (e.g., main, analytics).
List Schemas GET /api/2.1/unity-catalog/schemas Lists schemas within a specific catalog. Query param: ?catalog_name=main.
List Tables GET /api/2.1/unity-catalog/tables Lists tables within a schema. Query param: ?catalog_name=main&schema_name=dw.
Get Table Details GET /api/2.1/unity-catalog/tables/{full_name} Gets detailed metadata (columns, types, comments) for a specific table.

3. Transformation & SQL Execution

Goal: Preview SQL queries and validate logic in the "Visual Transformation Designer".

Operation Method Endpoint Description
Execute SQL POST /api/2.0/sql/statements Executes a SQL statement on a SQL Warehouse. Used for data preview and SQL validation.
Get Statement Result GET /api/2.0/sql/statements/{statement_id} Retrieves the results of an executed SQL statement.

4. Version Control (Databricks Repos)

Goal: Commit and push generated SQL/Code to the Git repository.

Operation Method Endpoint Description
Get Repo Info GET /api/2.0/repos/{repo_id} Checks the status of the connected repository (branch, head commit).
Update Repo PATCH /api/2.0/repos/{repo_id} Pulls the latest changes from the remote Git provider.
Write File POST /api/2.0/workspace/import Writes the generated SQL file to the specific path in the Workspace (Repo).

5. Jobs & Execution

Goal: Create, schedule, and monitor Databricks Jobs.

Operation Method Endpoint Description
List Jobs GET /api/2.1/jobs/list Populates the "My Created Jobs" table.
Create Job POST /api/2.1/jobs/create Creates a new job definition with the generated task.
Trigger Run POST /api/2.1/jobs/run-now Manually triggers a run of an existing job.
List Runs GET /api/2.1/jobs/runs/list Populates the "Recent Job Runs" table and "Monitoring" charts.
Get Run Output GET /api/2.1/jobs/runs/get-output Retrieves logs and output for a specific job run.

6. Job Failures & Debugging

Goal: Identify, analyze, and recover from job failures.

Operation Method Endpoint Description
List Failed Runs GET /api/2.1/jobs/runs/list List runs with ?status=FAILED to identify recent issues.
Get Run Output GET /api/2.1/jobs/runs/get-output Get error logs/stack traces. Query param: ?run_id={run_id}.
Repair Run POST /api/2.1/jobs/runs/repair Retry a failed run, optionally specifying a subset of tasks.
Cancel Run POST /api/2.1/jobs/runs/cancel Cancel a stuck or hanging job run.

7. Data Lineage

Goal: Visualize upstream and downstream dependencies.

Operation Method Endpoint Description
Get Table Lineage GET /api/2.0/lineage-tracking/table-lineage Retrieves lineage data for a specific table.

8. Monitoring & Analytics

Goal: Populate the dashboard charts.

Operation Method Endpoint Description
List All Runs GET /api/2.1/jobs/runs/list Used to aggregate success/failure rates and execution times.
